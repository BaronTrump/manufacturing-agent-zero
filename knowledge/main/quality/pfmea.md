# PFMEA — Process Failure Mode and Effects Analysis

## Methodology (AIAG-VDA Handbook)

### Step 1: Planning and Preparation
- Define scope (system, subsystem, component)
- Establish cross-functional team
- Use Process Flow Diagram as input

### Step 2: Structure Analysis
- Identify process steps (station-level)
- Identify process elements (man, machine, material, method, environment)
- Identify process characteristics (product vs. process)

### Step 3: Function Analysis
- Describe function of each process step
- Use "Verb + Noun + Measurable" format
- Link process functions to product characteristics

### Step 4: Failure Analysis
- Failure Mode (FM): How might the process fail?
- Failure Effect (FE): What happens to the customer?
- Failure Cause (FC): Why does the failure happen?
- Current Prevention Controls: What prevents the cause?
- Current Detection Controls: What detects the failure/cause?

### Step 5: Risk Analysis
- Severity (S): 1-10 rating of effect severity
- Occurrence (O): 1-10 rating of cause frequency
- Detection (D): 1-10 rating of detection effectiveness
- Action Priority (AP): High/Medium/Low (replaces RPN in AIAG-VDA)

### Step 6: Optimization
- Define recommended actions
- Assign responsibility and target completion date
- Implement actions and re-rate severity/occurrence/detection
- Validate effectiveness

### Step 7: Results Documentation
- Document FMEA results
- Communicate to relevant stakeholders
- Use as living document (update with changes)

## Severity Criteria (10-point scale)
- 10: Safety/regulatory — affects safe vehicle operation
- 9: Safety/regulatory — likely to cause customer health/safety hazard
- 8: Major disruption to plant (line shutdown > 10 min)
- 7: High customer dissatisfaction, major rework
- 6: Moderate customer dissatisfaction, rework off-line
- 5: Moderate customer dissatisfaction, rework in-station
- 4: Minor customer dissatisfaction, minor rework
- 3: Minor annoyance, easily handled
- 2: Very minor, noticed by discriminating customer
- 1: No effect
